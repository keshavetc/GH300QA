What are the different GitHub Copilot plans available?
GitHub Copilot plans (as of 2025):
  1. Copilot Individual:
    • $10/month or $100/year
    • For individual developers
    • Access to Copilot in IDEs, CLI, mobile
    • Monthly or annual billing
  2. Copilot Business:
    • $19/user/month (billed annually) or $21/user/month (monthly billing)
    • For organizations
    • Organization-wide policy management
    • Content exclusions
    • Audit logs
    • Minimum 1 seat, no maximum
  3. Copilot Enterprise:
    • $39/user/month
    • All Business features plus:
    • Personalized models (fine-tuned on your code)
    • Documentation search
    • Pull request summaries
    • Advanced security & compliance
Free access: Students, teachers, open source maintainers (verified)
How does billing work for Copilot Business?
Business billing details:
  • Pricing structure:
    • Per-seat pricing: $19/user/month (annual) or $21/user/month (monthly)
    • Charged per assigned seat, not total organization members
  • Billing cycle:
    • Monthly: Billed on the same day each month
    • Annual: Billed once per year, upfront
  • Proration:
    • Adding seats mid-cycle: Prorated charge for remainder of billing period
    • Removing seats: Credit applied to next billing cycle
  • Payment methods:
    • Credit card
    • Invoice (for larger organizations)
    • Purchase order (Enterprise)
Example: 50 seats √ó $19/month = $950/month (annual billing) or $1,050/month (monthly billing)

What happens if I cancel my Copilot subscription?
Cancellation process and effects:
  • Access:
    • Copilot access continues until end of current billing period

  • Data:

    • Can request data deletion per GDPR
  • Reactivation:
    • Can resubscribe at any time

How does licensing work for contractors and external collaborators?

How is Copilot usage tracked for billing purposes?
Usage tracking mechanisms:
  • Seat-based billing:
    • Charged per assigned seat, not actual usage
    • Doesn't matter if user uses Copilot 1 hour or 100 hours
    • Fixed cost per seat
  • No usage-based billing:
    • Unlike some AI services, no per-request charges
    • Unlimited suggestions within subscription
  • Seat assignment tracking:
    • Organization admins can see assigned users
    • Audit logs track seat grants/revocations
    • Billing portal shows seat count
Visibility: Admins can view seat assignments but not individual developer usage metrics (privacy)

What happens to billing when I change my organization size?
Billing adjustments for size changes:

Adding seats (scaling up):
  • Prorated charge for remainder of billing cycle
  • Example: Add 10 seats mid-month, pay ~$95 (half of $190 for 10 seats)
  • Next cycle: Full charge for all seats
Removing seats (scaling down):
  • Credit applied to next billing cycle
  • Unused time calculated proportionally
  • No immediate refund; offset future invoices
Large-scale changes:
  • Growing from 50 to 500 seats: Consider contacting Sales for better pricing
  • Downsizing significantly: Review contract terms (annual commitments)
Tip: Make seat adjustments early in billing cycle to minimize proration complexity

How do I manage Copilot licenses across multiple organizations?

Scenario: User belongs to multiple GitHub organizations

Licensing approach:
  • Separate licenses per org:
    • Each organization manages its own Copilot subscriptions
    • User can have access via multiple orgs
    • Each org bills for seats they assign
  • Individual subscription:
    • User maintains their own Individual subscription
    • Works across all organizations they belong to
    • User pays, not organizations
Management complexity:
  • Each org admin manages seats independently
  • No centralized cross-org license management (by design)
  • Users can switch which org's license they use
Best practice: Clarify which org provides Copilot for shared users
